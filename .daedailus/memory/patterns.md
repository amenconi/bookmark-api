# Established Patterns

## Data Access
- **Prisma ORM Pattern**: All database operations through Prisma client
- **Generated Types**: Database models auto-generated by Prisma
- **SQLite**: File-based database for development simplicity

## API Design
- **RESTful Endpoints**: Standard HTTP methods (GET, POST, PUT, DELETE)
- **Resource-based URLs**: /api/bookmarks, /api/tags structure
- **Express Router**: Modular route organization

## Configuration Management
- **Environment Variables**: dotenv for configuration
- **Separate Config Files**: Dedicated config directory
- **Type-safe Config**: TypeScript interfaces for configuration

## Error Handling
- **Centralized Middleware**: Dedicated error handling middleware
- **Express Error Handling**: Standard Express error handling patterns

## Application Structure
- **Layered Architecture**: Config → Middleware → Routes → Models
- **Separation of Concerns**: Each directory has specific responsibility
- **Entry Point Pattern**: Single index.ts as application entry

## Development Workflow
- **Hot Reload**: ts-node-dev for development
- **Build Process**: TypeScript compilation to dist/
- **Type Safety**: Strict TypeScript configuration

## Import Patterns
- **Path Aliases**: @/* for src/ directory imports
- **Module Resolution**: Node.js style module resolution
- **CommonJS**: require/module.exports pattern

## Examples
```typescript
// Typical Express route structure
app.get('/api/bookmarks', async (req, res, next) => {
  try {
    // Prisma database operation
    const bookmarks = await prisma.bookmark.findMany();
    res.json(bookmarks);
  } catch (error) {
    next(error); // Pass to error middleware
  }
});

// Configuration pattern
const config = {
  port: process.env.PORT || 3001,
  nodeEnv: process.env.NODE_ENV || 'development'
};
```
